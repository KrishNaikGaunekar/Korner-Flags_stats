- This connects the camera movement correction (written by CameraMovementEstimator.add_adjust_position_to_tracks as 'position_adjusted') to the perspective transform.

In camera_movement_estimator/camera_movement_estimator.py:
1. Line 65: Change `old_feature = cv2.goodFeaturesToTrack(frame_gray, **self.features)` to `old_features = cv2.goodFeaturesToTrack(frame_gray, **self.features)` (add the trailing 's' so it reassigns the variable used on line 49 by calcOpticalFlowPyrLK).

2. Lines 16-19 (mask construction in __init__): Replace the hardcoded mask columns with proportional values derived from the frame's actual width. The frame is already passed as the first argument to __init__. Change:
   ```python
   first_frame_grayscale = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
   mask_features = np.zeros_like(first_frame_grayscale)
   mask_features[:,0:20] = 1
   mask_features[:,900:1050] = 1
   ```
   To:
   ```python
   first_frame_grayscale = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
   h, w = first_frame_grayscale.shape[:2]
   mask_features = np.zeros_like(first_frame_grayscale)
   mask_features[:, 0:int(0.010 * w)] = 1
   mask_features[:, int(0.469 * w):int(0.547 * w)] = 1
   ```
   Rationale: 0/20 on 1920px = 0.0/0.010, 900/1920 = 0.469, 1050/1920 = 0.547. These proportions preserve the original calibration while adapting to any frame width.
  </action>
