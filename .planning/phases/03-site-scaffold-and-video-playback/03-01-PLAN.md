  <action>
1. Create `.github/workflows/` directory.

2. Create `.github/workflows/deploy.yml` with this exact content:

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]
    paths:
      - 'site/**'
      - 'data/**'
      - '.github/workflows/deploy.yml'
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Install, build, and upload site
        uses: withastro/action@v5
        with:
          path: ./site
          node-version: 22

  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

Key points:
- `permissions` block includes `pages: write` and `id-token: write` (prevents 403 errors)
- `concurrency` with `cancel-in-progress: false` (prevents race-clobbering deploys)
- `withastro/action@v5` with `path: ./site` (handles install + build + upload-pages-artifact internally)
- `node-version: 22` explicit (Astro 6 requires Node 22+)
- `paths` filter so only site/ or data/ changes trigger rebuilds
- `workflow_dispatch` for manual trigger from GitHub UI
- Two separate jobs: `build` and `deploy` (required by Pages API)
- Do NOT add `actions/configure-pages` or `actions/upload-pages-artifact` steps — `withastro/action@v5` handles these internally
