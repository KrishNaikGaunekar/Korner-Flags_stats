<decisions>
## Implementation Decisions

### Heatmap layout
- **Side-by-side two-column grid** — Team 1 left, Team 2 right
- Collapses to single column at **640px** breakpoint (same as existing responsive patterns)
- Images fill **full column width** (`width: 100%`) — mplsoccer pitch proportions scale cleanly
- No max-width cap needed

### Team labeling
- Each heatmap has a **colored heading above the PNG**: "Team 1" in blue, "Team 2" in orange
- Exact same blue/orange as Phase 4 possession bar (Team 1 = `#0071e3`, Team 2 = `#e8732a`)
- Label above → then PNG below (header-first, not caption)

### Section framing
- Section heading: **"Team Heatmaps"** (matches current placeholder heading)
- Add a one-line subtitle below the heading: **"Where each team spent most of their time on the pitch"**
- Same card treatment as the stats section: white bg, `border-radius: 12px`, `box-shadow: 0 1px 3px`

### AI disclaimer
- Small grey caption at the bottom of the section, same visual treatment as player table disclaimer
- Exact wording: **"AI-estimated from video — values are approximate"**

### Claude's Discretion
- Exact gap/spacing between the two heatmap columns (within the Apple-esque minimal aesthetic)
- Whether to use CSS Grid or Flexbox for the two-column layout
- `alt` text for each heatmap image (accessibility)
- Whether the Astro component is inline in `[slug].astro` or extracted to a `HeatmapSection.astro` component

</decisions>
